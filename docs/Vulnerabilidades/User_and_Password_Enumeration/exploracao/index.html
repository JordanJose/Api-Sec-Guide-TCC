Exploração

Tendo em mente os questionamentos levantados ao final da sessão anterior, é possível realizar os testes de segurança a fim de descobrir onde a vulnerabilidade pode ser explorada.

Um bom passo inicial é mapear e estudar os métodos de autenticação na aplicação, possibilitando conhecer o funcionamento e as possíveis fragilidades.

Comportamento de login na aplicação​

Para que um usuário faça login, é necessário que o mesmo se autentique na aplicação, para isso basta utilizar a rota /users/v1/login usando uma requisição do tipo POST enviando as informações de login no body da requisição.

Para realizar os testes de comportamento do login na aplicação, serão utilizados dois usuários:

  • Jordan - Existe e possui a senha password
  • Jose - Não existe

Analisando o comportamento de login​

Ao analisar o comportamento das mensagens e status de login, é possível comprovar que há uma possibilidade de enumerar senhas e nomes de usuário.

  • Só há sucesso quando o usuário consegue fazer login, usando um nome de usuário e senha corretos
  • A aplicação avisa quando um nome de usuário é inexistente, usando uma senha correta (existente no sistema) ou não.
  • A aplicação informa quando uma senha não é correta para um respetivo usuário.

As mensagens fornecidas são muito específicas, fornecendo detalhes desnecessários que podem ser usados pelo atacante que esteja testando uma lista de usuários e senhas na aplicação.
