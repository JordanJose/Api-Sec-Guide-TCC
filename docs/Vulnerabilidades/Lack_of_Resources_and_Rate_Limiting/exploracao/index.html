Exploração

Tendo em mente os questionamentos levantados ao final da sessão anterior, é possível realizar os testes de segurança a fim de descobrir onde a vulnerabilidade pode ser explorada.

Esse tipo de vulnerabilidade possui muitas maneiras de ser explorada, algumas são complexas e envolvem protocolos de redes, porém, também é interessante realizar testes de desempenho, carga e stress, para saber se a aplicação foi devidamente dimensionada.

Pegando o gancho da vulnerabilidade anterior, a enumeração de usuários e senhas, é interessante fazer o teste de limitação de requisições na rota de login, além de evitar um ataque DoS/DDoS, evitaria que a enumeração foi feita com facilidade.

Rate limit da rota de login​

Para que um usuário faça login, é necessário que o mesmo se autentique na aplicação, para isso basta utilizar a rota /users/v1/login usando uma requisição do tipo POST enviando as informações de login no body da requisição.

Dessa vez a intenção não é realizar o login, é apenas testar se há algum tipo de limitador de requisições em um curto espaço de tempo.

Para isso, não é necessário desenvolver nenhum script (o que seria interessante), o próprio Burp Suite possui algumas ferramentas para fazer esse tipo de teste.

Como a intenção é apenas realizar requisições, a ferramenta do Burp chamada sequencer pode ajudar nisso.

A intenção dessa ferramenta é analisar responses a fim de encontrar formas de prever strings, como tokens e rotas, porém, será utilizada apenas para checar se todas as requsições enviadas foram bem sucedidas.

Os passos são simples:

1 - Selecione a requisição que deseja repetir na aba target do Burp Suite

2 - Envie a requisição para o sequencer

3 - Com a requisição no sequencer, basta clicar em start live capture que uma nova janela com o teste em execução irá se abrir

4 - Basta aguardar os testes encerrarem para analisar os resultados, para esse teste é interessante apenas os números de requisições e erros

Analisando os resultados das requisições​

Ao analisar os números informados pelo Burp Suite e pelos logs da aplicação é possível perceber que foram realizadas 20 mil requisições em um espaço de tempo de aproximadamente 5 minutos, todas com sucesso.

Um usuário comum não conseguiria realizar essa quantidade absurda de tentativas de login em tão pouco tempo, porém, a aplicação permite que isso seja feito.

Ou seja, a aplicação está suscetível a tentativas de DoS/DDoS, além de outros tipos de vulnerabilidades que a falta de limitação de requisições está atrelada, como a enumeração de usuários e senhas.
